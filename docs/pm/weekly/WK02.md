## Blockers encountered

- **P50 miscalibration** -- conformal coverage landed at 0.28 instead of 0.50. Root cause is seasonal distribution shift between the calibration set (Aug-Oct) and test set (Oct-Dec). Documented as a known limitation with the production fix identified (rolling calibration window). P90 held.
- **Quantile model indentation bug** -- the training loop was at module level instead of inside the function body. Caught during notebook run, fixed in the same session.
- **Conformal alpha missing from function signature** -- `fit_conformal` used `alpha` in the body but it was not declared as a parameter. Fixed immediately.
- **Jupyter kernel cache** -- after editing `conformal.py`, the notebook continued using the cached version. Fix: Kernel -> Restart and Run All.

## What I learned

Built the full probabilistic forecasting chain from raw residuals to calibrated intervals. Pinball loss is the key insight: by penalising under- and over-prediction asymmetrically, a model can learn any percentile of a distribution without modelling the distribution explicitly. Conformal prediction then adds a mathematical guarantee that the stated coverage holds on unseen data, provided the calibration and test sets are drawn from the same distribution. The TimescaleDB layer shows this is not just a notebook project: data lives in a production-grade time-series database, and SQL window functions handle the feature engineering inside the storage layer.
