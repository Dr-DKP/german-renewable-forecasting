# Weekly Reports

> **Corporate vocab:** A *weekly report* (or *status update* / *status report*) is a short written document summarising what you committed to last week, what you actually delivered, what blocked you, and what is next. The *cadence* (writing one every week, on time) is often more signal than the content itself.

## Cadence for P2 (3 reports, not 6 — ruthless cut)

- **WK01** — end of Day 7 (2026-04-24) — covers Phases 0 + 1
- **WK02** — end of Day 14 (2026-05-03) — covers Phases 2-3 + start of 4
- **WK03** — end of Day 22 (2026-05-12) — covers Phases 5-6 (sprint close)

Claude drafts each report during the Day 7 / 14 / 22 Close block. User edits and commits.

## Template (copy to `WK0N.md`)

```markdown
# Weekly Report — WK0N

**Week of:** YYYY-MM-DD to YYYY-MM-DD
**Sprint days:** Day N to Day M
**Phase(s) covered:** <short names>

## What I committed to
- [x] Thing done
- [ ] Thing not done — why?

## What I actually delivered
(1-3 bullet points; link commits / PRs)

## Metrics
- Commits: N
- New wiki notes: N
- ADRs written: N
- MLflow runs logged: N (if applicable)
- Kanban cards closed: N

## Blockers encountered
(one line each — what, impact, resolution)

## What I learned — interview-flavoured
(one paragraph in English + 3-5 key German technical terms / Fachbegriffe)

## Plan for next week
(2-3 bullets)

## Risk register update
(list risks that status-changed since last report)
```
